Troubleshooting the Invalid Octal Number Error in JSR223 PreProcessors with TimeShift Function

Introduction


The JSR223 PreProcessor is a powerful tool for scripting in JMeter. It allows users to write scripts in a variety of languages, including JavaScript, Groovy, and Beanshell. One of the most useful functions of the JSR223 PreProcessor is the TimeShift function, which allows users to shift the time of a request by a specified amount. However, some users have reported that the TimeShift function throws an “Invalid octal number” error on some iterations only. In this article, we will discuss the cause of this error and how to fix it.


What is the TimeShift Function?


The TimeShift function is a powerful tool for scripting in JMeter. It allows users to shift the time of a request by a specified amount. This is useful for testing applications that require requests to be sent at specific times. For example, if an application requires requests to be sent at 10:00 AM, the TimeShift function can be used to shift the time of the request to 10:00 AM.


What Causes the Invalid Octal Number Error?


The “Invalid octal number” error is caused by a bug in the TimeShift function. The bug occurs when the TimeShift function is used with a negative value. When a negative value is used, the TimeShift function attempts to convert the value to an octal number, which is not supported. As a result, the TimeShift function throws an “Invalid octal number” error.


How to Fix the Invalid Octal Number Error


The “Invalid octal number” error can be fixed by using a positive value instead of a negative value when using the TimeShift function. For example, if the TimeShift function is used with a negative value of -1, it should be changed to a positive value of 1. This will ensure that the TimeShift function does not attempt to convert the value to an octal number, and the “Invalid octal number” error will not be thrown.
