Python lamba syntax. Why is it 'arguments' and not 'parameters?'

Python lambda syntax is a powerful tool for creating anonymous functions. It is often used in combination with other functions to create complex expressions. The syntax of a lambda expression is quite simple, consisting of the keyword lambda, followed by a comma-separated list of arguments, and then a colon and the expression to be evaluated.


The question of why the arguments in a lambda expression are referred to as arguments and not parameters is an interesting one. The answer lies in the fact that the arguments in a lambda expression are not actually parameters, but rather are used to define the scope of the expression.


When a function is defined, the parameters are used to define the scope of the function. That is, the parameters are used to determine what values the function will accept as input and what values it will return as output. In contrast, the arguments in a lambda expression are used to define the scope of the expression itself. That is, the arguments are used to determine what values the expression will evaluate and what values it will return as output.


In other words, the arguments in a lambda expression are not actually parameters, but rather are used to define the scope of the expression. This is why they are referred to as arguments and not parameters.


The use of arguments in a lambda expression can be quite powerful. For example, a lambda expression can be used to create a function that takes multiple arguments and returns a single value. This is often used in combination with other functions to create complex expressions.
