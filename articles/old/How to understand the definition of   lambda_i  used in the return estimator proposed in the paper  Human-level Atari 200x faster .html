How to understand the definition of $\lambda_i$ used in the return estimator proposed in the paper "Human-level Atari 200x faster"?

The paper “Human-level Atari 200x faster” proposes a return estimator that uses the concept of $\lambda_i$ to measure the expected return of a given state. Understanding the definition of $\lambda_i$ is essential to understanding the return estimator and how it works.


In the paper, $\lambda_i$ is defined as the “discounted sum of future rewards”. This means that $\lambda_i$ is a measure of the expected return of a given state. It is calculated by summing up the discounted rewards of all future states that can be reached from the current state. The discount factor is used to reduce the importance of rewards that are further away in the future.


To calculate $\lambda_i$, the paper proposes a recursive formula. This formula is based on the idea that the expected return of a given state is equal to the immediate reward plus the discounted expected return of the next state. This formula can be expressed as follows:


$\lambda_i = r_i + \gamma \lambda_{i+1}$


where $r_i$ is the immediate reward for the current state and $\gamma$ is the discount factor.


The paper also proposes a “bootstrapping” approach to calculating $\lambda_i$. This approach is based on the idea that the expected return of a given state can be estimated by using the expected return of the next state. This approach can be expressed as follows:


$\lambda_i \approx r_i + \gamma \hat{\lambda}_{i+1}$


where $\hat{\lambda}_{i+1}$ is the estimated expected return of the next state.
