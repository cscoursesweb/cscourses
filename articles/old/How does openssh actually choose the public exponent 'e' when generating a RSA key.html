How does openssh actually choose the public exponent 'e' when generating a RSA key?

OpenSSH is a widely used open source implementation of the Secure Shell (SSH) protocol. It is used to securely connect to remote systems, transfer files, and execute commands. One of the key components of the SSH protocol is the use of public-key cryptography, which relies on the generation of a public/private key pair. The public key is shared with the remote system, while the private key is kept secret.


When generating a RSA key, OpenSSH must choose a public exponent ‘e’. This is an important parameter in the RSA algorithm, as it determines the strength of the encryption. The public exponent ‘e’ must be a positive integer that is relatively prime to the totient of the modulus. The modulus is the product of two large prime numbers, and the totient is the number of positive integers less than the modulus that are relatively prime to the modulus.


OpenSSH typically uses a public exponent ‘e’ of 65537. This is a commonly used value, as it is a Fermat prime and is considered to be a safe choice. It is also a relatively small number, which makes it faster to use than larger exponents.


When generating a RSA key, OpenSSH also allows the user to specify a different public exponent ‘e’. This is useful if the user wants to use a larger exponent for increased security. However, it is important to note that larger exponents can significantly slow down the encryption process.
