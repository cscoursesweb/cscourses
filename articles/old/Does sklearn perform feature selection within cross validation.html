Does sklearn Perform Feature Selection Within Cross Validation?


The short answer is yes, sklearn does perform feature selection within cross validation. Cross validation is a technique used to evaluate the performance of a machine learning model on unseen data. It is a popular method for assessing the accuracy of a model and is often used in combination with feature selection. Feature selection is the process of selecting the most relevant features from a dataset to use in a model.


In sklearn, feature selection is performed within cross validation using the Recursive Feature Elimination (RFE) algorithm. RFE is a feature selection algorithm that works by recursively removing features from the dataset until the desired number of features is reached. The algorithm works by first fitting a model to the dataset and then evaluating the importance of each feature. The least important features are then removed from the dataset and the model is re-fit. This process is repeated until the desired number of features is reached.


The advantage of using RFE within cross validation is that it allows for the selection of the most relevant features while also evaluating the performance of the model on unseen data. This ensures that the model is not overfitting to the training data and is able to generalize to unseen data.
